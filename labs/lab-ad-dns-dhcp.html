Lab – Active Directory / DNS / DHCP

Ce lab a pour objectif de mettre en place un environnement Windows Server permettant la gestion centralisée des utilisateurs, la résolution DNS et l’attribution automatique des adresses IP.

Contexte : Formation TSSR
Durée : 1 journée
Environnement : Windows Server 2022, Windows 10, virtualisation

Architecture du lab

  • 1 contrôleur de domaine (AD, DNS, DHCP)
  • 1 poste client Windows 10
  • Réseau : 192.168.10.0/24
Schéma réseau du lab (à remplacer par une image)

Étapes principales

  1. Installation du rôle Active Directory Domain Services
  2. Promotion du serveur en contrôleur de domaine
  3. Configuration du DNS intégré à l’AD
  4. Mise en place d’une plage DHCP
  5. Jointure du poste client au domaine

Tests et vérifications

  • Le poste client reçoit automatiquement une adresse IP
  • Résolution DNS fonctionnelle avec nslookup
  • Connexion réussie avec un compte utilisateur du domaine
Capture ipconfig / nslookup (à remplacer)

Difficultés rencontrées

  • Le client ne recevait pas d’IP → serveur DHCP non autorisé dans l’AD
  • Problème de résolution DNS → mauvaise configuration DNS côté client

Compétences mises en œuvre

  • Administration Active Directory
  • Gestion des services DNS et DHCP
  • Diagnostic réseau
  • Virtualisation d’environnement de test
