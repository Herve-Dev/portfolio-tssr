Lab – Déploiement Active Directory

Ce lab a pour objectif de mettre en place un contrôleur de domaine Active Directory afin de centraliser l’administration des utilisateurs et des ordinateurs dans un environnement Windows.

Contexte : Formation TSSR
Environnement : Windows Server 2025, virtualisation VMware
Réseau : 192.168.10.0/24 (NAT)

Architecture du lab

  • 1 serveur Windows Server 2025
  • Rôles : Active Directory Domain Services, DNS, DHCP
  • Domaine : lab.local
  • 1 poste client Windows 11
  • Réseau : NAT VMNet8 – serveur en IP statique 192.168.10.20, client en IP dynamique

Le lab est volontairement simple afin de se concentrer sur la mise en place et la compréhension d’Active Directory.

Étapes réalisées

  1. Installation de Windows Server 2025
  2. Configuration réseau avec IP statique pour le serveur
  3. Installation et configuration du rôle Active Directory Domain Services
  4. Installation et configuration du rôle DNS intégré à l’AD
  5. Installation et configuration du rôle DHCP, création d’une plage et exclusion de l’IP du serveur
  6. Jointure du poste client au domaine

Contrôleur de domaine Active Directory

Le serveur a été promu en contrôleur de domaine. Le domaine lab.local est opérationnel et permet la gestion centralisée des ressources via Active Directory.

DNS intégré à Active Directory

Le rôle DNS a été installé et configuré pour le domaine lab.local. Les zones sont intégrées à Active Directory et les enregistrements SRV assurent la résolution des services critiques pour le domaine.

DHCP – Attribution automatique des adresses IP

Le rôle DHCP a été installé et autorisé dans Active Directory. Une plage d'adresses a été configurée pour distribuer automatiquement les IP aux clients du domaine lab.local, avec exclusion de l'IP du serveur pour éviter tout conflit.

Compétences mises en œuvre

  • Installation et administration d’Active Directory
  • Configuration réseau d’un serveur Windows
  • Virtualisation d’un environnement de test
  • Documentation technique d’un lab
